📋 八百万プロジェクト 変数名簿（2026/01/20 版）
1. 基盤・インフラ系 (Global / Context)
アプリの土台を支える重要な変数たちです。

ctx: egui::Context。画面描画の司令塔。

ui: 各パネル内で描画を行うためのペン。

fonts: フォント設定用。

visuals: アプリの見た目（テーマ）設定用。

response: クリックやドラッグの反応を検知。

2. 言語処理・解析系 (Lexer / Parser / Evaluator)
プログラムを解析して動かす「心臓部」の変数です。

input: 解析前の文字列（ソースコード）。

position: 現在どこを読んでいるかの位置。

ch: 現在読んでいる1文字。

ident: 識別子（変数名や命令名）。

start: 読み取り開始位置。

レキシカ / パーサ / 実行機: 各処理フェーズのインスタンス。

プログラム / 構文木: 解析し終わったデータの塊。

first_token / op (operator): 命令の頭文字や演算子。

3. 命令・ロジック内変数 (Internal Logic)
条件分岐や計算の中で一時的に使われる変数です。

cmd / 内側の命令 / 実行内容: 実行されるコマンド群。

条件 / 内部 / 内部の式: もし〜なら 等の判定用。

左辺 / 右辺 / 左 / 右: 比較や計算の両端。

値 / s: 具体的なデータや文字列。

buffer / output: 文字列を組み立てる際の一時保管場所。

4. ファイル・エディタ管理系 (File System / UI)
繭さんが操作する「ファイル」に関連する変数です。

current_file: 今まさに編集しているファイル。

path: ファイルの保存場所。

content: ファイルの中身（テキスト）。

name: タブに表示される名前。

label: UI上の表示用テキスト。

line_count / line_numbers: 行番号表示用。

layouter / layout_job: 構文ハイライトの描画指示。

5. 色管理・定数 (Color Theme)
hex_color (関数): 16進数から色を作る魔法。

hex_str: #FFFFFF などの入力文字列。

r, g, b: 赤・緑・青の成分。

color: 最終的な色のデータ。
